1 The Terminal Window

All computers come with a native terminal window (also commonly called a command-line shell). If you are unfamiliar with the terminal or how to find it on your computer, see the page On Getting a Terminal Window.


Now try a series of commands in your terminal. The $ represents your information that the terminal prints automatically (like your username or the host name). At the prompt, exter:

$ whoami

You should be told your user name, e.g.:

caitlynpickens

Next, try an automated build:

$ make --version

A version number should be printed:

GNU MAKE 3.81


You have now completed the first software setup test!

Test 2 <-->

2 Text Editors for Code

In this boot camp, you will be writing your own code, which requires a text editor. The choice of which editor to use is up to you. Some suggestions are: nano, VI, Emacs, and XEmacs. Decide which editor you want to use, and then make sure it is installed by attempting to create a new file using that editor or checking the version (as shown below). If the editor is not yet installed, follow the links provided above to the editor's website and follow their installation instructions.

nano:

nano is designed as a free replacement for the Pico text editor. To see if you have nano installed, enter the following command at the prompt:

$ nano --version

You should be given a version number, e.g.:

GNU nano version 2.0.6 (compiled 10:55:37, May 25 2011)

VI:

VIM is a highly configurable text editor designed to enable efficient text editing. To see if you have VIM installed, enter the following command at the prompt:

$ vi --version

You should be given a version number, e.g.:

VIM - Vi IMproved 7.3 (2010 Aug 15, compiled Feb 21 2012 18:42:47)

Emacs:

Emacs is an open-source text editor. To see if you have Emacs installed, enter the following command at the prompt:

$ emacs --version

You should be given a version number, e.g.:

GNU Emacs 22.1.1

XEmacs:

XEmacs is the next-generation version of Emacs. It is extremely customizable. To see if you have XEmacs installed, enter the following command at the prompt:

$ xemacs --version

You should be given a version number, e.g.:

-bash: xemacs: command not found

This error message appeared because XEmacs is not yet installed. If I want to use XEMacs as my primary text editor, I still need to install it. After install, I test the version again:

XEmacs 21.5 (beta31) "ginger"

3 Python for Scripting

Any Python 2 version will suffice for this class. We will not be teaching Python 3, so please make sure that you have either Python 2.6 or 2.7 versions.

To check that it's installed and what version you have, at the prompt, enter:

        $ python --version

You should be given a version number, e.g.:

        Python 2.7.3

Now, check its behavior:

        $ python
        >>> print 2+2
        4

And, check it has the sqlite3 library if your boot camp is covering relational databases:

        > import sqlite3

If all is fine it won't complain.


If any portion of this test did not pass, then you probably do not have Python 2.6 or later installed. See On Installing Python for installation guidelines.

Python setuptools for installing Python packages

At the prompt, enter:

    $ easy_install --version

You should be given a version number, e.g.:

    distribute 0.6.10

or a warning:

    usage: easy_install [options] requirement_or_url ...
    or: easy_install --help
    error: option --version not recognized

Python nose for running unit tests

At the prompt, enter:

    $ nosetests --version

You should be given a version number, e.g.:

    nosetests version 0.10.4

At the prompt, enter:

    $ nosetests

You should be given a test report, e.g.:

    ........
    Ran 0 tests in 0.003s
    OK

BitBucket for using remote version control repositories

For version control practicals, we use BitBucket to enable repositories to be used remotely and to be shared. To get a BitBucket account for version control practicals, just sign up for free at https://bitbucket.org/.

Mercurial for distributed version control

At the prompt, enter:

    $ hg --version

You should be given a version number, e.g.:

    Mercurial Distributed SCM (version 1.4)

http://mercurial.selenic.com/

EasyMercurial for version control via a GUI

  • Create a directory (folder) somewhere on your computer.
  • Start EasyMercurial either by selecting it from a menu, double-clicking an icon or running: 
                $ ./EasyMercurial
    at the command-line, depending upon your operating system
  • Open Repository dialog will appear
  • Click File folder
  • Click Browse... and select your directory
  • Click OK

http://easyhg.org/

SQLite for a lightweight relational database

At the prompt, enter:

    $ sqlite3 -version

You should be given a version number, e.g.:

    3.6.20

http://www.sqlite.org/

Firefox for the SQLite manager below

Depending upon your operating system either start up Firefox from its menu or icon, or, at the prompt, enter:

    $ firefox

http://www.mozilla.org/en-US/firefox/new/

Firefox SQLite manager for a relational database

Once Firefox is running:

  • Select Tools => SQLite Manager
  • SQLite Manager window appears
  • Select Database => New database

http://code.google.com/p/sqlite-manager/
